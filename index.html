Why

내가 마주한 상황

의심 문자가 왔을 때,
부모님이 혼자 판단해야 하는 순간.

  1. 1부모님 폰에 모르는 문자가 옵니다.
  2. 2지금 당장 자녀에게 물어보기 어렵습니다.
  3. 3앱 설치·가입이 필요한 서비스는 쓰기 어렵습니다.
  4. 4그래서 붙여넣기 한 번으로 끝나게 만들었습니다.
01

사기범은 불안과 다급함을 무기로 씁니다

"지금 안 하면 큰일 난다"는 압박이 정상적인 판단을 흔듭니다.

02

위급한 순간엔 자녀에게 물어볼 생각조차 안 납니다

패닉이 오면 "일단 확인해보자"는 여유 자체가 사라집니다.

03

기존 확인 수단은 문턱이 너무 높습니다

설치·회원가입·계좌 연결이 필요한 순간, 정작 필요한 분은 멈춥니다.

교육과의 연결

리부트 AI 과정에서 배운 국내 LLM 활용과 프롬프트 설계를, 부모님 문자 확인이라는 실제 생활 문제에 바로 적용했습니다.

What

붙여넣기 → 판정 → 공유.
세 번이면 충분합니다.

문턱 없음

설치·가입·로그인 없이 링크 하나로 바로 사용. 홈 화면에 추가하면 앱처럼 켤 수 있습니다.

어르신 눈높이

큰 글씨, 쉬운 말, 결과 읽어주기, 위험 부분 하이라이트.

혼자 끝내지 않기

가족 공유와 공식 상담 안내(1394·1332)로 다음 행동을 연결합니다.

작은 공익 한 줄

확인 후 남는 자리에는 광고 대신 경찰청 실종 공개정보를 둡니다.

How Solar

그냥 불러 쓰지 않고,
판정 기준을 직접 설계했습니다.

역할 구분
메인: Upstage Solar Pro 3 Vercel Functions PWA Vanilla JS

국내 LLM이 메인입니다. Solar가 문자 위험도를 판정하고, Claude Code·Codex는 구현·QA 보조로만 사용했습니다.

5유형
링크형 사칭·협박형 기관 콜백형 가족 긴급송금형 링크 미끼형
안전 규칙
  • 입력에 없는 내용은 지어내지 않는다
  • 애매하면 낮음이 아니라 중간으로 판단한다
  • 의심 이유와 위험도는 반드시 일치해야 한다
BEFORE

판정 모순 발견

이유: "계좌 이체를 요구합니다" → 위험도: 낮음
근거는 사기인데 판정이 어긋나던 문제.

AFTER

규칙 추가 후

같은 문자 → 위험도: 높음
"이유와 위험도는 일치" 규칙을 넣어 바로잡았습니다.

발견 2

평범한 문자를 사기로 오판 → "지어내지 않기" 규칙으로 해결

발견 3

실종 사진 base64 원문 문제 → data URI 변환으로 정상 표시
